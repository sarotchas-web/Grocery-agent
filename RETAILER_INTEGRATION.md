# חיבור קמעונאים

## המצב הנוכחי

הפורטל תומך בהזנה ידנית של הצעות סל עדכניות. הוא אינו מחובר כרגע לחשבון או ל־API של קמעונאי, ואינו מבצע רכישה או תשלום.

לחיבור חי יש להשתמש רק ב־API רשמי או באינטגרציה שאושרה בכתב על ידי הקמעונאי. אין לשמור בקוד, ב־Git, ביומנים או בכתובות URL סיסמאות, אסימוני גישה, פרטי תשלום או כתובת משלוח מלאה.

## מה צריך לקבל מהקמעונאי

1. תיעוד API רשמי וסביבת בדיקות.
2. כתובת בסיס של ה־API.
3. שיטת אימות מאושרת, רצוי OAuth.
4. הרשאות לקריאת קטלוג, זמינות, מחירים, מבצעים ודמי שירות.
5. הרשאות לקבלת אפשרויות משלוח ונקודות איסוף בזמן אמת.
6. כללים לגבי מגבלות קצב, תפוגת אסימונים ושמירת נתונים.

## מימוש המתאם

יש ליצור מחלקה נפרדת לכל קמעונאי שמממשת את `RetailerAdapter` מתוך `src/grocery_agent/retailer_adapter.py`:

```python
class MyRetailerAdapter:
    def quote_delivery(
        self,
        basket_id: str,
        delivery_profile_id: str,
    ) -> OrderEstimate:
        ...

    def list_pickup_points(
        self,
        basket_id: str,
    ) -> tuple[PickupPoint, ...]:
        ...
```

`quote_delivery` חייב להחזיר:

- זמינות של כל מוצר.
- מחירי מוצרים.
- הנחות ומבצעים.
- דמי משלוח ושירות.
- הערכת משקל למוצרים שקילים.

`list_pickup_points` חייב למשוך בזמן ההשוואה:

- נקודות איסוף פעילות.
- שם ואזור של כל נקודה.
- חלונות איסוף זמינים.
- דמי איסוף.

מנוע המדיניות יסנן אוטומטית נקודות שאינן בעמק חפר, חלונות שאינם חופפים ל־16:30–18:30, וסלים שחסרים בהם מוצרים.

## סודות וכתובת משלוח

- יש לטעון סודות רק ממשתני סביבה מקומיים.
- אין להעביר סוד בפרמטר URL.
- אין להדפיס בקשות, כותרות אימות או תגובות גולמיות ביומן.
- יש להשתמש במזהה `delivery_profile_id` בתוך המערכת.
- אם API המשלוח דורש כתובת מלאה, יש לפענח אותה רק ברגע הבקשה המקומית ולשלוח אותה ב־HTTPS בגוף הבקשה בלבד.
- אין לשמור את הכתובת המפוענחת במטמון, בקובץ זמני או בתוצאת API פנימית.

## לפני הפעלה חיה

יש להוסיף בדיקות חוזה מול סביבת הבדיקות של הקמעונאי עבור:

- סל מלא וסל עם מוצר חסר.
- חישוב מחיר לאחר הנחות וכל העמלות.
- מוצר שקיל.
- נקודות איסוף ואזורים.
- חלונות זמן.
- תפוגת אסימון ושגיאות רשת.
- היעדר כתובת, סודות ופרטי תשלום ביומנים ובתגובות.

לאחר קבלת פרטי ה־API הרשמיים של קמעונאי מסוים ניתן לממש ולרשום את המתאם שלו בלי לשנות את מנוע התקציב, האיסוף או ההמלצות.
